# Claims and Figure Dependency Audit

This document reconstructs the paper claim registry from `paper/claims/recompute_all.py`, `paper/claims/scripts/compute_*.py`, the manuscript sources in `paper/sources/**`, and the publication figure modules in `src/paper_figures/**`. It intentionally does not rely on the excluded generated `paper/claims/claims.json`.

Verdict: the claim layer is mostly recomputable from PRIMARY restricted artifacts (`data/raw_responses/**` and `data/splits/**`) plus retained code, with local database access for ICD labeling-function comparisons. The figure layer is not fully self-contained from primary artifacts: selected figures parse regenerated `codex_outputs/*`, `data/optimization/*`, `data/production/parquet/*`, or regenerated `paper/claims/claims.json` intermediates.

## Reproduction Path

A credentialed user starts from their own MIMIC-IV v3.1 and MIMIC-IV-Note v2.2 database load, configured as described in `SCHEMA_ASSUMPTIONS.md`. The minimal path is not a single turnkey command; it is the ordered project path below.

1. Build split artifacts with `scripts/02_build_note_splits.py`, `scripts/19_build_1k_split.py`, `scripts/25_build_5k_split.py`, and the extended-sample path in `scripts/46_run_extended_5k.py`. These regenerate `data/splits/*.csv` and `data/splits/SPLITS_MANIFEST.json`.
2. Run extraction batches with the relevant split and prompt/model settings: refinement/holdout, methodology 1k, methodology 5k audit, production, paired/full-model, extended, and reasoning comparison runs. The retained drivers include `scripts/05_validate_on_holdout.py`, `scripts/06_run_1k_smoke.py`, `scripts/07_run_5k_methodology.py`, `scripts/08_run_full_production.py`, `scripts/31_launch_production.sh`, `scripts/40b_retry_failed.py`, `scripts/45_run_gold_standard.py`, `scripts/55_run_paired_gold.py`, `scripts/56_run_reasoning_comparison.py`, and `scripts/46_run_extended_5k.py`. These regenerate `data/raw_responses/<run_id>/results.jsonl`.
3. Regenerate kappa and generalization sidecars with `scripts/14_threeway_kappa.py`, `scripts/17_refinement_vs_holdout.py`, `scripts/22_kappa_3way_comparison.py`, `scripts/26_kappa_4way_comparison.py`, and `scripts/46b_extended_kappa.py`. These provide the `codex_outputs/*kappa*.json` files required by stability/generalization figure loaders.
4. Regenerate load-bearing intermediate reports with `scripts/34_prelaunch_confusion_analysis.py`, `scripts/41_export_parquet.py`, `scripts/55b_paired_analysis.py`, `scripts/91_paired_collapse_analysis.py`, `scripts/92_disagreement_collapse_analysis.py`, `scripts/93_audit_cluster_collapse.py`, and optionally `scripts/96_paired_bootstrap_ci.py`. These recreate report/parquet/audit-corpus inputs that selected figures or diff checks read.
5. Recompute the claim registry locally with `python paper/claims/recompute_all.py`, then verify with `python paper/claims/verify_claims.py`. This creates a local `paper/claims/claims.json`; that generated file is intentionally excluded from the release because the original copy contained hashes and absolute paths for restricted artifacts.
6. Build figures with `python scripts/build_paper_figures.py` and the manuscript with `python paper/build.py` as needed. Figure generation writes to local `docs/figures/paper/**`; build outputs remain generated artifacts and are not shipped.

Primary-data path: most numeric claims are recomputed directly from `data/raw_responses/**` and `data/splits/**`. Required-intermediate path: some final figures and paper-prep code require regenerated reports under `codex_outputs/**`, `data/optimization/**`, `data/production/parquet/**`, or the local regenerated `paper/claims/claims.json`.
